Capture Attribute

The capture attribute is used in HTML forms to specify that a file input should directly open the device's camera or microphone when activated, allowing users to capture media (like photos or audio) on the spot. This attribute is particularly useful for mobile devices, enhancing user experience by streamlining the process of uploading media files.

he capture attribute specifies that, optionally, a new file should be captured, and which device should be used to capture that new media of a type defined by the accept attribute. Values include user and environment. The capture attribute is supported on the file input type. The capture attribute takes as its value a string that specifies which camera to use for capture of image or video data, if the accept attribute indicates that the input should be of one of those types.

Example

<input type="file" accept="image/*" capture="environment">

In this example, when the user interacts with the file input, it will open the device's rear camera to capture an image.

Note: The capture attribute is a hint to the browser, and its behavior may vary depending on the device and browser support. Not all browsers or devices may honor this attribute, and users may still have the option to select files from their device storage instead of capturing new media.
