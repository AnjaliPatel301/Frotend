Iframe

The <iframe> tag specifies an inline frame. An inline frame is used to embed another document within the current HTML document.

The <iframe> tag is often used to insert content from another source, such as an advertisement, a video, or a web page, into a web page.

Note: The <iframe> tag is not supported in HTML 4.01 Transitional.

Attributes

The <iframe> tag supports the following attributes:

  • src: Specifies the URL of the page to embed
  • width: Specifies the width of the iframe
  • height: Specifies the height of the iframe
  • name: Specifies the name of the iframe
  • sandbox: Enables an ext ra set of restrictions for the content in the iframe
  • allow: Specifies a feature policy for the iframe
  • loading: Specifies whether to load the iframe immediately or to defer loading until it reaches a certain distance from the viewport
  • referrerpolicy: Specifies which referrer information to send when fetching the iframe content
  • title: Provides a title for the iframe, which is used for accessibility purposes

Note

By default, if the embedded page is from a different origin (domain, protocol, or port) than the parent page, the embedded page cannot access the parent page's DOM (Document Object Model) due to security restrictions known as the Same-Origin Policy. This means that scripts running in the embedded page cannot interact with or manipulate the content of the parent page.

However, there are ways to enable communication between the embedded page and the parent page, such as using the postMessage API, which allows for safe cross-origin communication.

It's important to note that the Same-Origin Policy is a security feature implemented by web browsers to prevent malicious scripts from accessing sensitive information on other websites. Therefore, it's essential to be cautious when embedding content from external sources and to ensure that the embedded content is trustworthy.

Browser Support

The <iframe> tag is supported in all major browsers.
