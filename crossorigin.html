Autocomplete Attribute

The crossorigin attribute is used in HTML to handle Cross-Origin Resource Sharing (CORS) requests for certain elements, such as <img>, <script>, <link>, and <video>. It specifies how the element should behave when fetching resources from a different origin (domain) than the one that served the web page.

The crossorigin attribute can take the following values:

  • anonymous: This value indicates that the request for the resource should be made without credentials (like cookies or HTTP authentication). If the server does not include the appropriate CORS headers, the resource will not be loaded.
  • use-credentials: This value indicates that the request for the resource should be made with credentials. The server must include the appropriate CORS headers to allow this; otherwise, the resource will not be loaded.
  • omit: This value indicates that no CORS request should be made, and the resource will be fetched without any CORS handling. This is typically used when you want to avoid CORS checks altogether.

Example usage:

<img src="https://imgv3.fotor.com/images/videoImage/wonderland-girl-generated-by-Fotor-ai-art-generator.jpg" crossorigin="anonymous">

In this example, the image will be fetched from "https://example.com/image.jpg" without sending any credentials. If the server does not allow cross-origin requests, the image will not be displayed.

It's important to note that the crossorigin attribute is only relevant for resources that are fetched from a different origin. If the resource is on the same origin as the web page, the crossorigin attribute has no effect.
